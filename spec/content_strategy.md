# Content Strategy

- Bot generated from prompt.
    - Content will be from a narrow distribution.
    - Need new prompt each time to mitigate
    - Doesn't create value if a user could have prompted the bot themselves. (e.g. search for "top 10 blenders" and you
      get a splog article where the prompt was "top 10 blenders")
- Summarize/Comment on Article
    - Borderline content stealing (especially if the post is monetized and the original is no loner able to)
    - But also this is how real humans amplify an article and drive traffic to the journalist/creator
    - Ethical if it tracks and drive up click thru rates. Otherwise it is preventing traffic.
- Audience prompts the bot for content
    - This is RAG if it sticks to a document
    - This function calling if it uses some tool
    - This is a system prompt (an OpenAI style "GPT") or character card if that is all.
    - Generic chatbot interface if the bot is unconstrained (the harness is the feature.)
- Human content
    - Enriched with AI addons.
    - At some % of AI work, this becomes Bot generated content (see above)
- Human editing
    - Creates an exhausting amount of text to edit. Problems are stylistic, can't be copyedited like human text (that
      is stylistically fine but have punctation and spelling errors )
    - Fact checking is a lot of work
